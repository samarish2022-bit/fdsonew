# Анализ отступов между разделами

## Текущая схема

### Базовые правила (десктоп)

| Элемент | Отступ | Комментарий |
|--------|--------|-------------|
| **.section** | `padding: 5rem 0` | Общий вертикальный отступ для всех секций (сверху и снизу по 5rem). |
| **.hero** | `padding: 6rem 0 2.5rem` | Сверху 6rem, снизу 2.5rem (переход к Новостям). |
| **.section-news** | переопределяет только `padding-top: 1rem` | Уменьшает зазор между Hero и блоком «Новости». Остаётся `5rem` снизу от .section. |
| **.section-competitions** | `padding-top: 2.5rem` | Свой верх; низ — от .section (5rem). |
| **.section-photo** | `padding-top: 2.5rem` | Аналогично. |
| **.section-contacts** | без своего padding | Только фон; вертикаль полностью от .section (5rem). |
| **.section-documents** | не задан отдельно в базе | Использует только .section (5rem сверху и снизу). |

Итог на десктопе:
- Между разделами по сути **5rem** (от .section): сверху у каждой секции 5rem, снизу тоже 5rem, т.е. «между» двумя секциями получается 5rem + 5rem = **10rem** по вертикали между контентом соседних блоков (если не переопределено).
- Исключения: Hero снизу 2.5rem; у Новостей сверху 1rem (значит Hero → Новости: 2.5rem + 1rem = 3.5rem); у Соревнований и Фото сверху 2.5rem (значит предыдущая секция снизу 5rem + 2.5rem = 7.5rem между контентом).

Из-за разных `padding-top` у секций **отступы между разделами визуально неодинаковые**:
- Hero → Новости: меньше (2.5 + 1 = 3.5rem).
- Новости → Соревнования: 5 + 2.5 = 7.5rem.
- Дальше между Соревнования / Фото / Документы / Контакты: то 5+2.5, то 5+5 в зависимости от того, переопределён ли у следующей секции `padding-top`.

### Планшеты (max-width: 767.98px)

| Элемент | Отступ |
|--------|--------|
| **.hero** | `padding: 2rem 0 2rem` |
| **.section** | `padding: 2rem 0` |
| **.section-news, .section-competitions, .section-photo, .section-documents** | `padding-top: 2rem; padding-bottom: 2rem` |
| **.section-contacts** | `padding-top: 2rem; padding-bottom: 2.5rem` |

Здесь отступы между разделами **единые**: 2rem сверху и снизу у каждой секции (Контакты снизу 2.5rem).

### Маленькие экраны (max-width: 575.98px)

| Элемент | Отступ |
|--------|--------|
| **.section** | `padding: 1.5rem 0` |
| **.section-news … .section-documents** | `padding-top: 1.5rem; padding-bottom: 1.5rem` |
| **.section-contacts** | `padding-top: 1.5rem; padding-bottom: 2rem` |

Отступы ещё меньше, но тоже выровнены между секциями (1.5rem, у Контактов снизу 2rem).

---

## Выводы

1. **Десктоп:** отступы между разделами **не единые**: разный `padding-top` у Новостей (1rem), Соревнований и Фото (2.5rem), у Документов и Контактов — наследуется 5rem от .section. Нижний отступ везде от .section (5rem), поэтому «зазор» между двумя секциями складывается из нижнего padding одной и верхнего другой и получается разным.
2. **Мобильные:** отступы сделаны едиными (2rem или 1.5rem) за счёт переопределения и для .section, и для каждой секции.
3. **.section-documents** на десктопе не имеет своего правила — использует только базовый .section; визуально между Документами и Контактами будет 5rem + 5rem.

---

## Рекомендации для единообразия

- **Вариант A (единый зазор на десктопе):** задать у всех секций одинаковый вертикальный padding, например только через .section (оставить `padding: 5rem 0`) и **убрать** переопределения `padding-top` у .section-news, .section-competitions, .section-photo. Тогда между любыми двумя разделами будет одинаковый визуальный зазор.
- **Вариант B (оставить акценты):** оставить меньший отступ после Hero (как сейчас), но для остальных секций задать один и тот же `padding-top` и `padding-bottom` (например 2.5rem или 3rem), чтобы зазоры между Новости / Соревнования / Фото / Документы / Контакты были одинаковыми.

При необходимости можно вынести значение «отступ между разделами» в переменную, например `--section-gap: 3rem`, и использовать её в .section и при переопределениях.

## Отступ перед названием раздела

### В чём проблема

На десктопе отступ **перед** заголовком секции («Новости», «Соревнования» и т.д.) задаётся только **padding-top** у `.section` — сейчас **5rem** (при 16px базе это ~80px). Это воспринимается как слишком большой «пустой» зазор над названием.

У самого `.section-title` своего `margin-top` нет, поэтому весь зазор над заголовком = `padding-top` секции.

### Варианты решения

| Вариант | Действие | Результат |
|--------|----------|------------|
| **1. Уменьшить общий padding секции** | Задать `.section { padding: 3rem 0 }` (вместо 5rem) | Отступ перед названием = 3rem; зазор между секциями = 3rem + 3rem = 6rem. Всё визуально меньше и ровнее. |
| **2. Ещё компактнее** | `.section { padding: 2.5rem 0 }` | Перед названием 2.5rem, между секциями 5rem. |
| **3. Разный верх/низ** | `.section { padding: 2.5rem 0 4rem }` | Мало места над заголовком (2.5rem), больше — после контента (4rem). Зазор между секциями = 4 + 2.5 = 6.5rem. |

Рекомендация: **вариант 1** (3rem) — и отступ перед названием, и зазоры между разделами уменьшаются, но остаются читаемыми. При желании можно заменить 3rem на 2.5rem (вариант 2).

## Мобильная версия — отступы между разделами

### Текущее состояние (актуальные правила в style.css)

**Планшеты и телефоны (max-width: 767.98px):**
- **.hero:** `padding: 2rem 0 2rem` — снизу 2rem перед первой секцией.
- **.section:** переопределён на `padding: 2rem 0` (верх и низ по 2rem).
- **.section-news, .section-competitions, .section-photo, .section-documents:** `padding-top: 2rem; padding-bottom: 2rem`.
- **.section-contacts:** `padding-top: 2rem; padding-bottom: 2rem`.
- **.section-documents + .section-contacts:** `padding-top: 0` (зазор Документы → Контакты = 2rem).

**Итог для 767px:** зазор между двумя секциями = padding-bottom предыдущей + padding-top следующей = **2rem + 2rem = 4rem**. Отступ перед названием раздела = **2rem**. Исключение: между Документами и Контактами зазор **2rem** (т.к. у Контактов сверху 0).

**Очень узкие экраны (max-width: 575.98px):**
- **.hero:** `padding: 1.5rem 0 1.5rem`.
- **.section:** `padding: 1.5rem 0`.
- **.section-news … .section-documents:** `padding-top: 1.5rem; padding-bottom: 1.5rem`.
- **.section-documents:** дополнительно `padding-bottom: 2rem`.
- **.section-documents + .section-contacts:** `padding-top: 0`.

**Итог для 575px:** зазор между секциями = **1.5rem + 1.5rem = 3rem**. Перед названием = **1.5rem**. Между Документами и Контактами = **2rem**.

### Несогласованность с десктопом

На десктопе зазор между разделами и отступ перед названием сделаны **2rem** (padding-top 2rem, padding-bottom 0). На мобильных:
- на 767px зазор между разделами **4rem** (вдвое больше), перед названием 2rem;
- на 575px зазор **3rem**, перед названием 1.5rem.

То есть на планшетах/телефонах (767px) отступы между блоками визуально крупнее, чем на десктопе.

### Рекомендации

| Вариант | Действие | Результат |
|--------|----------|------------|
| **A. Унифицировать зазор 2rem на 767px** | В медиазапросе 767px: у секций задать `padding-top: 2rem; padding-bottom: 0` (как на десктопе). Оставить `.section-documents { padding-bottom: 2rem }` и `.section-documents + .section-contacts { padding-top: 0 }`. | Зазор между разделами = 0 + 2rem = **2rem**, отступ перед названием = **2rem**. Едино с десктопом. |
| **B. Оставить 4rem на 767px** | Ничего не менять. | Зазоры остаются крупнее на планшетах. |
| **C. Унифицировать и на 575px** | В медиазапросе 575px: у секций `padding-top: 1.5rem; padding-bottom: 0`, у `.section-documents` — `padding-bottom: 2rem`, контакты после документов — `padding-top: 0`. | Зазор между разделами = **2rem** (за счёт padding-bottom у предыдущей или 2rem у документов), перед названием = **1.5rem**. Либо задать везде 2rem (padding-top 2rem, padding-bottom 0, у документов bottom 2rem) для полного совпадения логики с десктопом на узких экранах. |

**Рекомендация:** вариант **A** — в брейкпоинте 767px перевести секции на схему «padding-top: 2rem; padding-bottom: 0», сохранив специальные правила для Документы/Контакты. Тогда на планшетах и телефонах зазор между разделами и отступ перед названием будут такими же, как на десктопе (2rem).
